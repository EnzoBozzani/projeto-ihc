# Planejamento de Usabilidade (método DECIDE)

| Etapa | Descrição |
|-------|-----------|
| **D** | - Avaliar se os usuários conseguem criar agentes de forma autônoma e sem dificuldades. <br> - Avaliar se os usuários conseguem construir fluxos utilizando os agentes criados. <br> - Avaliar se os usuários conseguem compreender e utilizar a interface de teste de fluxo. <br> - Avaliar se o desempenho (tempo de resposta e estabilidade) dos fluxos e agentes é adequado à experiência do usuário. <br> - Avaliar a usabilidade do processo de integração de fontes externas (banco de dados, PDFs, etc.). |
| **E** | - Os usuários conseguem criar agentes sem dúvidas quanto aos campos obrigatórios e opções disponíveis? <br> - Os usuários conseguem criar fluxos completos utilizando os agentes configurados? <br> - Os usuários conseguem identificar e compreender as ações necessárias para testar um fluxo na interface de teste? <br> - Os usuários entendem e interpretam corretamente os resultados e logs da execução dos fluxos? <br> - O desempenho do sistema (tempo de resposta e estabilidade) interfere na realização das tarefas? <br> - Os usuários conseguem integrar fontes externas (Banco de Dados, PDFs, etc.) sem erros ou confusão nos passos? |
| **C** | - Teste de usabilidade com tarefas (observação direta da execução de atividades). <br> - Entrevistas semiestruturadas para captar dificuldades e percepções. <br> - Grupo focal para discussão coletiva das barreiras e oportunidades de melhoria. |
| **I** | - **Recrutamento de usuários:** entre 5 e 10 participantes com familiaridade básica com ferramentas digitais. <br> - **Equipamentos:** computadores com navegador atualizado e acesso à internet; ambiente adequado para observação. <br> - **Cronograma:** <br> &nbsp;&nbsp;• *Semana 1:* recrutamento, agendamentos, preparação dos roteiros e protótipos. <br> &nbsp;&nbsp;• *Semana 2:* execução das sessões, entrevistas, questionários e grupo focal. <br> - **Recursos de apoio:** roteiros de tarefas, formulários, protótipos navegáveis, ambiente preparado para registro. |
| **D** | - Práticas alinhadas à LGPD com termo de consentimento informado. <br> - Dados anonimizados e usados exclusivamente para fins de pesquisa e melhoria do sistema. <br> - Participantes podem desistir a qualquer momento sem prejuízo. <br> - Não serão coletados dados sensíveis além do necessário; gravações apenas com autorização explícita. |
| **E** | - Análise combinando dados quantitativos e qualitativos para apoiar decisões de refinamento. <br> - **Quantitativos:** tempo de tarefa, taxa de sucesso, erros, frequência de uso (estatística descritiva). <br> - **Qualitativos:** percepções, dificuldades, sugestões e barreiras analisadas por categorização temática. <br> - Resultados apresentados em tabelas, gráficos e sínteses narrativas, permitindo triangulação das descobertas. |

# Lista de Instrumentos

i) Termo de consentimento informado (LGPD)  
ii) Roteiro de tarefas para teste de usabilidade  
iii) Roteiro de entrevista semiestruturada  
iv) Tabela de observação das sessões  
v) Formulário para grupo focal  
vi) Formulário de avaliação heurística
