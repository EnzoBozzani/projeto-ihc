# Descrição do Procedimento de Preparação do teste

## **1. Definição dos Objetivos da Avaliação**

O objetivo da avaliação é verificar se um usuário real consegue utilizar corretamente as funcionalidades principais do sistema FluxusAI. Para este teste, espera-se que o usuário seja capaz de:

- Criar quatro agentes distintos, incluindo um agente orquestrador e três agentes especialistas com funções diferentes.
- Iniciar um novo projeto, entendendo o fluxo de navegação entre as telas.
- Conectar os agentes no editor visual, estabelecendo corretamente as ligações que definem o fluxo.
- Executar o fluxo criado, interpretando o retorno do sistema na aba de testes.

Esses objetivos permitem avaliar não apenas a facilidade de navegação, mas também a clareza dos feedbacks, a compreensão do fluxo e a eficiência do sistema.

---

## **2. Seleção dos Participantes Representativos**

Para este teste, foi selecionado um usuário que trabalha em uma empresa de pequeno/médio porte que realiza processos de cobrança baseados em planilhas e manipulação de dados. Trata-se de um perfil semelhante ao público-alvo da ferramenta, que tende a lidar com automações simples para tarefas administrativas.

Esse tipo de usuário representa bem o cenário de aplicação do FluxusAI, pois possui conhecimento intermediário em informática, mas não necessariamente experiência prévia com agentes ou fluxos orquestrados.

---

## **3. Preparação dos Cenários e das Tarefas Realistas**

O cenário proposto reproduz uma situação realista de trabalho:

> O usuário trabalha em uma empresa que possui clientes assinantes de um produto. Alguns clientes estão com faturas atrasadas e é necessário criar uma automação que consulte o banco de dados, identifique esses clientes inadimplentes e envie automaticamente um e-mail de cobrança para cada um deles.

Com base nisso, as tarefas do teste serão:

1. Criar um agente orquestrador.  
2. Criar um agente para buscar dados no banco SQL.  
3. Criar um agente para processar dados.  
4. Criar um agente para enviar e-mails.  
5. Criar um novo projeto no FluxusAI.  
6. Adicionar cada agente ao fluxo e conectá-los corretamente.  
7. Executar o fluxo no ambiente de testes e verificar o resultado.

Cada tarefa foi elaborada para refletir uma operação real e mensurável, permitindo observar erros, hesitações e compreensão geral do sistema.

---

## **4. Configuração do Ambiente de Teste**

O ambiente foi preparado para garantir uma sessão de avaliação consistente:

- O usuário terá acesso à **versão mais atual** e estável do projeto FluxusAI.  
- Será utilizado um **computador de mesa** com monitor e mouse.  
- Um **gravador de tela** ficará ativo durante todo o teste (com consentimento).  
- O avaliador estará presente, mas não interferirá — apenas observará e registrará.

Esse ambiente assegura que toda a interação seja registrada e que não haja interferências externas durante o teste.

---

## **5. Executar os Critérios de Avaliação**

Os critérios que serão observados durante o teste incluem:

- Tempo de conclusão de cada tarefa  
- Número e tipo de erros cometidos  
- Hesitações, dúvidas ou comentários espontâneos  
- Necessidade de pedir ajuda  
- Navegação correta entre telas  
- Clareza no entendimento dos feedbacks do sistema  
- Capacidade de interpretar o resultado da execução do fluxo

O avaliador seguirá um roteiro padronizado para garantir que todos os participantes sejam avaliados pelas mesmas métricas.

---

## **6. Registrar Dados**

Durante o teste, serão registrados:

- Tempos de execução por tarefa  
- Sucesso ou insucesso em cada etapa  
- Erros cometidos (categorizados)  
- Prints ou trechos relevantes do vídeo de gravação  
- Observações comportamentais (expressões, falas, hesitações)  
- Notas do avaliador sobre dificuldades encontradas  
- Respostas ao formulário pós-teste (percepção de facilidade, satisfação, etc.)

Esses dados permitirão gerar um relatório de avaliação completo e fundamentado.
